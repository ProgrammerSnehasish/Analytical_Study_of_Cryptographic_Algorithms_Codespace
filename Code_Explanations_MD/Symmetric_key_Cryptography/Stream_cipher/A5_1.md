# A5_1 Code
```python
class LFSR:
    def __init__(self, length, taps, clock_bit):
        self.register = [0] * length
        self.taps = taps
        self.clock_bit = clock_bit

    def feedback(self):
        return sum(self.register[tap] for tap in self.taps) % 2

    def shift(self):
        fb = self.feedback()
        self.register = [fb] + self.register[:-1]

    def get_clock_bit(self):
        return self.register[self.clock_bit]

def majority(a, b, c):
    return (a & b) | (a & c) | (b & c)

def a5_1_initialize(key_bits):
    R1 = LFSR(19, [13, 16, 17, 18], 8)
    R2 = LFSR(22, [20, 21], 10)
    R3 = LFSR(23, [7, 20, 21, 22], 10)

    # Load key bits (64-bit key)
    for i in range(64):
        bit = key_bits[i]
        R1.register[0] ^= bit
        R2.register[0] ^= bit
        R3.register[0] ^= bit
        R1.shift()
        R2.shift()
        R3.shift()

    return R1, R2, R3

def a5_1_keystream(R1, R2, R3, n_bits):
    stream = []
    for _ in range(n_bits):
        m = majority(R1.get_clock_bit(), R2.get_clock_bit(), R3.get_clock_bit())
        if R1.get_clock_bit() == m:
            R1.shift()
        if R2.get_clock_bit() == m:
            R2.shift()
        if R3.get_clock_bit() == m:
            R3.shift()
        z = R1.register[-1] ^ R2.register[-1] ^ R3.register[-1]
        stream.append(z)
    return stream

def str_to_bits(s):
    return [int(b) for byte in s.encode('utf-8') for b in f"{byte:08b}"]

def bits_to_str(bits):
    return bytes(int("".join(map(str, bits[i:i+8])), 2) for i in range(0, len(bits), 8)).decode('utf-8', errors='ignore')

def xor_bits(a, b):
    return [x ^ y for x, y in zip(a, b)]

# === User Input ===
key_input = input("Enter a 64-bit key (8 characters): ")
plaintext_input = input("Enter the plaintext to encrypt: ")

if len(key_input) != 8:
    raise ValueError("Key must be exactly 8 characters (64 bits).")

key_bits = str_to_bits(key_input)
plaintext_bits = str_to_bits(plaintext_input)

# A5/1 encryption
R1, R2, R3 = a5_1_initialize(key_bits)
ks = a5_1_keystream(R1, R2, R3, len(plaintext_bits))
cipher_bits = xor_bits(plaintext_bits, ks)

# Decryption (same process, stream cipher)
R1_d, R2_d, R3_d = a5_1_initialize(key_bits)
ks_d = a5_1_keystream(R1_d, R2_d, R3_d, len(cipher_bits))
decrypted_bits = xor_bits(cipher_bits, ks_d)

# === Output ===
print("\nEncrypted (binary):", ''.join(map(str, cipher_bits)))
print("Encrypted (hex):", hex(int(''.join(map(str, cipher_bits)), 2))[2:])
print("Decrypted text:", bits_to_str(decrypted_bits))

```

## 🧩 Overview

The code simulates A5/1 Stream Cipher, a synchronous stream cipher used to encrypt and decrypt data bit by bit using pseudorandom keystream bits generated from Linear Feedback Shift Registers (LFSRs).

Encryption and decryption both work as:

Ciphertext=Plaintext⊕Keystream
Plaintext=Ciphertext⊕Keystream

where ⊕ denotes bitwise XOR.

## ⚙️ Algorithm Components
### 1. Linear Feedback Shift Register (LFSR)
Class Definition
```python
class LFSR:
    def __init__(self, length, taps, clock_bit):
        self.register = [0] * length
        self.taps = taps
        self.clock_bit = clock_bit
```

An LFSR is a shift register that generates pseudorandom sequences.
Each register cell stores one bit (0 or 1), and when it shifts, a new bit (called feedback bit) is computed using XOR of certain tap positions.

Feedback and Shift:
```python
def feedback(self):
    return sum(self.register[tap] for tap in self.taps) % 2

def shift(self):
    fb = self.feedback()
    self.register = [fb] + self.register[:-1]
```

**Feedback bit**: XOR of specific bits (at taps) in the register.

**Shift**: Register shifts right; new bit enters on the left.

**🧮 Time Complexity (per shift)**:
Feedback = O(#taps) → usually ≤ 4 → O(1)
Shift = O(L), where L = register length (since list slice [:-1] copies).
So total per shift: O(L)

**📦 Space Complexity (per register)**: O(L)

### 2. Majority Function
```python
def majority(a, b, c):
    return (a & b) | (a & c) | (b & c)
```

Computes the majority bit (most common of 3 bits).
Used to control which LFSRs get clocked (shifted) at each step.

**🧮 Time: O(1) 📦 Space: O(1)**

### 3. Initialization (Key Loading)
```python
def a5_1_initialize(key_bits):
    R1 = LFSR(19, [13, 16, 17, 18], 8)
    R2 = LFSR(22, [20, 21], 10)
    R3 = LFSR(23, [7, 20, 21, 22], 10)

    for i in range(64):
        bit = key_bits[i]
        R1.register[0] ^= bit
        R2.register[0] ^= bit
        R3.register[0] ^= bit
        R1.shift()
        R2.shift()
        R3.shift()
```

Purpose: Load a 64-bit key into all three registers sequentially.
Each bit of the key is XORed into the leftmost bit of each LFSR, followed by a shift.

**📘 Algorithm (simplified)**:

Start with all registers = 0.

For each key bit:

XOR it into each LFSR.

Shift all LFSRs once.

**🧮 Time Complexity**:

64 iterations × (3 shifts per iteration × O(L))
= O(64 × L) = O(L) (since L is constant small, 19+22+23)

**📦 Space Complexity: O(19+22+23) = O(1) constant space**.

### 4. Keystream Generation
```python
def a5_1_keystream(R1, R2, R3, n_bits):
    stream = []
    for _ in range(n_bits):
        m = majority(R1.get_clock_bit(), R2.get_clock_bit(), R3.get_clock_bit())
        if R1.get_clock_bit() == m: R1.shift()
        if R2.get_clock_bit() == m: R2.shift()
        if R3.get_clock_bit() == m: R3.shift()
        z = R1.register[-1] ^ R2.register[-1] ^ R3.register[-1]
        stream.append(z)
    return stream
```

**Algorithm Logic:**

Compute the majority bit of 3 “clock bits.”

Only shift the LFSRs whose clock bit matches the majority bit.

Compute keystream bit z as XOR of last bits of all registers.

Append z to output.

**⏳ Time Complexity:**

Each iteration (1 keystream bit):

3 bit reads + majority (O(1))

Up to 3 shifts (O(L))

1 XOR and append (O(1))

So per bit = O(L), and for n_bits → O(n × L)
(but L is small and constant ⇒ effectively O(n))

**📦 Space Complexity:**

Registers: O(1)

Keystream list: O(n)
→ O(n) total space.

### 5. Encryption and Decryption
cipher_bits = xor_bits(plaintext_bits, ks)
decrypted_bits = xor_bits(cipher_bits, ks_d)

Since A5/1 is a synchronous stream cipher, encryption and decryption are identical — XOR with the same keystream.

C=P⊕K,P=C⊕K

**🧮 Time Complexity**: O(n)
**📦 Space Complexity**: O(n)

### 6. Utility Functions

str_to_bits() — converts string to bit array (O(n))

bits_to_str() — converts bits to string (O(n))

xor_bits() — element-wise XOR (O(n))

All are linear time in message length.

| Stage | Operation | Time Complexity | Space Complexity |
|:------|:-----------|:----------------|:-----------------|
| Key Initialization | Load 64-bit key | O(L) | O(1) |
| Keystream Generation | Generate n keystream bits | O(n × L) ≈ O(n) | O(n) |
| Encryption / Decryption | XOR with keystream | O(n) | O(n) |
| **Total** | — | **O(n)** | **O(n)** |

*(L = max length of LFSR, constant ≈ 23)*

## 🧠 Conceptual Summary (Algorithm Flow)

Algorithm: A5/1 Stream Cipher

Input: 64-bit key, plaintext bits
Output: Ciphertext bits (same length)

Steps:

Initialize 3 LFSRs of fixed lengths (19, 22, 23).

Load Key (64 bits):
XOR each bit into all LFSRs → Shift all → Repeat 64 times.

Generate Keystream:

Compute majority of clock bits (8th, 10th, 10th).

Shift only LFSRs matching the majority bit.

Output XOR of last bits.

Encrypt/Decrypt:
XOR plaintext/ciphertext with the keystream.

## 🔐 Example Flow (Simplified)

Suppose key = "ABCDEFGH" (8 bytes = 64 bits)
Plaintext = "HELLO"

Convert both to bits.

Initialize LFSRs with key bits.

Generate 40 keystream bits (for "HELLO" = 5×8 bits).

XOR plaintext bits with keystream → ciphertext.

To decrypt: regenerate the same keystream and XOR again.

## ⚡ Practical Note

This is a didactic (educational) implementation of A5/1 — not cryptographically secure today.
Modern cryptanalysis has broken A5/1 in seconds with precomputed tables.

Registers: Constant small O(1)

Deterministic: Yes — same key & input always yield identical stream.
